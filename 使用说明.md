待办事项系统技术文档

项目概述
本项目是一个基于前后端分离架构的轻量级“待办事项（Todo List）”管理系统。前端采用 Vue 3 + Vite 构建，后端使用原生 PHP 8.1 提供 RESTful API，数据库为 MySQL，支持通过 Docker 容器化部署至 Railway 平台，并具备自动初始化数据库能力。

在线地址
- 前端演示：（Vercel 部署地址）
- 后端 API：https://vueai-production.up.railway.app/api.php

方式一：直接使用（无需部署）

修改前端 vue2/src/App.vue 中的 API 地址：
const API_URL = 'https://vueai-production.up.railway.app/api.php';

然后运行前端：
cd vue2
npm install
npm run dev

注意：数据存在共用数据库中

方式二：自己部署一套（推荐）

准备
- GitHub 账号
- Railway 账号（后端+数据库）
- Vercel 账号（前端，可选）

步骤

1. 部署后端 API
1. Fork 本仓库到 GitHub
2. 登录 Railway → New Project → 选择你的仓库
3. 添加 MySQL 插件（Add Plugin → MySQL）
4. 在服务的 Variables 中添加：
      MYSQLHOST     = {{ MySQL.MYSQLHOST }}
   MYSQLPORT     = {{ MySQL.MYSQLPORT }}
   MYSQLUSER     = {{ MySQL.MYSQLUSER }}
   MYSQLPASSWORD = {{ MySQL.MYSQLPASSWORD }}
   MYSQLDATABASE = {{ MySQL.MYSQLDATABASE }}
   
5. 等待部署完成

2. 部署前端
1. 前端代码在 vue2/ 目录
2. 修改 vue2/src/App.vue 中的 API_URL 为你自己的后端地址
3. 部署到 Vercel（Import Git Repository → 选择 vue2 目录）

项目流程

1. 环境初始化：本地开发需安装 Node.js (>=18.0.0) 和 PHP 环境；生产环境通过 Dockerfile 自动构建包含 PHP 8.1、MySQL 客户端及必要扩展的镜像。
2. 数据库配置：后端通过环境变量（如 MYSQLHOST, MYSQLUSER 等）读取数据库连接信息，而非硬编码在代码中，确保安全性。首次启动时，start.sh 脚本会自动检测并导入 SQL 备份文件以初始化表结构。
3. 前后端交互：前端通过 HTTP 请求（Fetch API）调用后端的 api.php。后端解析请求参数（Action），执行相应的 SQL 操作并返回 JSON 格式数据。
4. 版本控制：使用 Git 管理代码，.gitignore 排除了 node_modules 等无关文件。针对跨平台协作，配置了 .gitattributes 强制统一换行符为 LF，避免 Windows/Linux 混用时的冲突。

文件夹与文件作用详解

根目录 (vue_ai/)
文件名   类型   作用说明
api.php   PHP   核心后端入口。处理所有前端的 API 请求（增删改查），包含数据库连接逻辑和业务分发。
config.template.php   PHP   数据库配置模板。定义了连接数据库所需的变量结构（Host, User, Pass 等），实际使用时需重命名或配置环境变量。
composer.json   JSON   PHP 依赖管理文件。目前内容为空 {}，表示暂未引入第三方 PHP 库（如 Laravel 等），仅使用原生 PHP。
Dockerfile   Config   容器构建脚本。定义了如何从 php:8.1-cli 基础镜像构建应用环境，包括安装 MySQL 驱动、复制代码、暴露端口等。
start.sh   Shell   启动脚本。在 Docker 容器启动时运行，负责等待 MySQL 就绪、自动导入 SQL 数据、最后启动 PHP 内置服务器。
dump-test_db...sql   SQL   数据库备份文件。包含 tasks 表的结构定义和初始测试数据。
.gitignore   Config   Git 忽略规则。防止将 node_modules、环境变量文件等上传到仓库。
使用说明.md   Markdown   项目文档，记录如何安装、运行及注意事项。

前端目录 (vue2/)
文件名/夹   作用说明
src/App.vue   主组件。包含了待办事项的 HTML 结构、CSS 样式以及核心的 JavaScript 逻辑（数据获取、添加、删除）。
src/main.js   Vue 应用的入口文件，负责挂载 App 组件。
package.json   前端依赖清单。定义了项目名称、版本、脚本命令（dev, build）以及依赖库（Vue, Pinia, Vite 等）。
vite.config.js   Vite 构建工具的配置文件。
vercel.json   Vercel 平台的部署配置文件（虽然项目主要部署在 Railway，但这可能是遗留或备选配置）。
.gitattributes   换行符策略文件。设置 * text=auto eol=lf，强制所有文本文件在 Git 中使用 Unix (LF) 换行符，解决你在 Windows 上看到的警告问题。

API 接口详细说明

后端接口统一入口为 api.php，通过 URL 参数 action 区分不同操作。所有接口均支持跨域访问（CORS）。

1. 获取任务列表
- URL: api.php?action=list
- 方法: GET
- 功能: 查询数据库中所有任务，按 ID 倒序排列。
- 返回示例:
[
  { "id": 1, "title": "我的第一个任务", "status": 0 },
  { "id": 2, "title": "学习 PHP", "status": 1 }
]

2. 添加新任务
- URL: api.php?action=add
- 方法: POST
- 参数 (Body JSON):
  - title (string, 必填): 任务标题。
- 逻辑: 接收 JSON 数据，插入数据库，默认状态为 0 (未完成)。
- 返回示例:
{ "message": "添加成功", "id": 3 }

3. 更新任务状态
- URL: api.php?action=update
- 方法: POST
- 参数 (Body JSON):
  - id (int, 必填): 任务 ID。
  - status (int, 必填): 新状态 (0 或 1)。
- 逻辑: 根据 ID 更新 is_completed 字段。
- 返回示例:
{ "message": "更新成功" }

4. 删除任务
- URL: api.php?action=delete
- 方法: POST
- 参数 (Body JSON):
  - id (int, 必填): 要删除的任务 ID。
- 逻辑: 根据 ID 物理删除数据库记录。
- 返回示例:
{ "message": "删除成功" }
